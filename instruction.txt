инструкция по запуску приложения DB_handler.

исполняемый файл "DB_handler.jar"

варинты запуска программы:

	- программа поддерживает 2 типа операций :
		- search
		- stat

		тип операции должен быть укзаан при запуске в качестве первого параметра. 
		после указаня типа вводятся имена файла источника( далее INPUT) и целевого файла( далее OUTPUT)
		формат данных в файлах - json
		стандартно - input.json и output.json

		Содержимое файла INPUT должно различаться в зависимости от типа операции согласно требованиям для данного приложения

		для типа search:

		Виды критериев с примерами из требований:

			- поиск покупателей с определенной фамилией. :
				{"lastName": "Иванов"}
			- поиск покупателей, купивших  определенный товар не менее, чем указанное число раз : 
				{"productName": "box", "minTimes": 2}
			- поиск покупателей, у которых общая стоимость всех покупок за всё время попадает в интервал : 
				{"minExpenses": 112, "maxExpenses": 4000}	
			- поиск покупателей, купивших меньше всего товаров. Возвращается не более, чем указанное число покупателей :
				{"badCustomers": 3}

			критерии должны содержаться в блоке "criterias"

		Содержимое файла OUTPUT при выходных данных подобного типа соответсвует примеру в требованиях к приложению

		для типа stat:

		Во входном файле передаётся интервал дат сбора статистики. В виде двух дат укзанных по тегам "startDate" и "endDate" соответсвенно.
		Формат даты: "yyyy-MM-dd".
		В выходном файле будет содержаться выборка за период из двух дат, где для каждого покупателя будет указан список всех уникальных
		товаров, приобретенных за указанный период, упорядоченных по суммарнй стоимости по убыванию.

	Вне зависимости от типа операции программа запускается в двух режимах :
		- с указанем данных БД
		- без указания.

		при указании данных, во время запуска необходимо ввести данные для БД после указания файла output в следующем порядке:
			- адрес
			- порт
			- имя бд
			- имя пользователя
			- пароль	
		 пример запуска с указанем данных:

		java -jar DB_handler.jar search input.json output.json localhost 5432 purch_DB postgres Ywk3lila

		соответсвенно, при отсутствии указания данных БД, используются прописанные в программе стандартные, а именно указанные выше данные.

		Важно! Во входном файле не допускается использование комментариев.

		База данных создается посредством выполнения скрипта CreatePurch_db.sql в папке с проектом.

		рабочая папка с проектом - DB_handler_jar.

		внутри - примеры input файла для типов операций search и stat, файл CreatePurch_db.sql, пустой output.json.
